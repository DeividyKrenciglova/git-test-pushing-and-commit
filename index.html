Git & GitHub

Version Control for Everyone

What is Git?

Git is a version control system that tracks changes to your files over time. Think of it like a detailed save system for your code - you can save snapshots of your project at any point and go back to previous versions if something goes wrong.

Why use Git?
  • Never lose your work - every change is saved
  • Work on different versions simultaneously
  • See exactly what changed and when
  • Easily undo mistakes

What is GitHub?

GitHub is a cloud platform that hosts your Git repositories online. While Git manages your files locally on your computer, GitHub stores a copy on the internet so you can:

  • Access your code from any computer
  • Share code with collaborators
  • Showcase your projects to others
  • Discover open-source projects

How It Works

The basic workflow looks like this:

1. Make changes → 2. Stage files → 3. Commit → 4. Push

Key Concepts:

  • Repository (Repo): A folder that contains your project and its history
  • Commit: A snapshot of your changes - like saving a game checkpoint
  • Branch: A separate line of development - great for trying new features
  • Push: Send your commits to GitHub
  • Pull: Download changes from GitHub to your computer
  • Merge: Combine changes from different branches

Why It's Useful for Teams

Git and GitHub shine when working with others:

Collaboration Made Easy
  • Multiple people can work on the same project without overwriting each other's changes
  • Branches let team members work on features independently, then merge when ready
  • Pull Requests allow code review before merging - catch bugs early
  • Issues & Projects help organize tasks and track progress
  • History shows who changed what, making it easy to debug

No more emailing code files back and forth or asking "who has the latest version?" - GitHub keeps everyone in sync!

Basic Commands

  • git init - Start a new repository
  • git clone - Copy a repository from GitHub
  • git add - Stage changes for commit
  • git commit -m "message" - Save your changes
  • git push - Upload to GitHub
  • git pull - Download latest changes
  • git branch - Create or view branches
  • git merge - Combine branches
